2 · Lógica Upsert (identificación del lead)

El sistema detecta automáticamente si el lead ya existe. Si existe → actualiza. Si no → crea contacto + lead. Los campos enviados se fusionan con los existentes (no se sobreescriben con null).

1
internal_uuid recomendado Identificador único de Comuro. Si está presente, tiene prioridad absoluta.
2
phone + project fallback Busca contacto por teléfono y lead por campaña/proyecto. Útil si no se tiene uuid.
3
external_id fallback Se busca como thread_id almacenado previamente.

6 · Consideraciones técnicas
  • Los campos se fusionan con los existentes — nunca se sobreescribe con null
  • Se pueden enviar actualizaciones parciales del mismo lead en múltiples llamadas
  • El campo context es clave para que el equipo de ventas entienda el interés
  • thread_id permite trazabilidad completa entre Comuro y el CRM
  • internal_uuid es el identificador más confiable — siempre incluirlo
  • El lead creado queda en etapa NUEVO en el pipeline del CRM
  • El origen del contacto se registra automáticamente como WEB
  • La API Key se configura en el CRM bajo Configuración → API Keys
